QR Code Model

This document describes the JSON model to use for Unitag’s QR Code API.

JSON model

Below is the global structure of a JSON object representing a QR Code:

{
    "content":
    {
        ...
    }, 
    "design":
    {
        ...
    }, 
    "owner":"20"
}

  • owner: a field used to match an existing user in the PostgreSQL database.

  • content: an object to define what is encoded in the QR Code.

  • design: an object to define the graphical properties of the QR Code.

Content object

The content object defines the data encoded in the QR Code and can only be one of the following types: TEXT, URL, CALL, GEOLOC, SMS, WIFI, VCARD, EMAIL, EMAIL, CALENDAR, CALL.

  • TEXT
{
    "type":"TEXT",
    "text":"Hello World"
}
  • URL
{
    "type":"URL",
    "url":"http://www.unitag.io"
}
  • GEOLOC
{
    "type":"GEOLOC",
    "geoloc":
    {
        "latitude":"43-36-16.2N",
        "longitude":"1-26-38.4E"
    }
}
  • SMS
{
    "type":"SMS",
    "sms":
    {
        "tel":"+33970805343",
        "message":"Hello World"
    }
}
  • WIFI
{
    "type":"WIFI",
    "wifi":
    {
        "security":"WPA",
        "ssid":"unitag_WIFI",
        "password":"your_password_here"
    }
}
  • VCARD
{
    "type":"VCARD",
    "vcard":
    {
        "firstname":"Romain",
        "lastname":"Kassel",
        "emailwork":"romain.kassel@unitag.io",
        "emailhome":"romain.kassel@gmail.com",
        "telcell":"+33970805341",
        "telwork":"+33970805342",
        "telhome":"+33970805343",
        "telfax":"+33970805344",
        "website":"http://www.unitag.io",
        "title":"CIO",
        "organization":"Unitag",
        "address":"67 allées Jean Jaurès",
        "postalcode":"31000",
        "city":"Toulouse",
        "country":"France"
    }
}
  • EMAIL
{
    "type":"EMAIL",
    "email":
    {
        "to":"romain.kassel@unitag.io",
        "cc":"contact@unitag.io",
        "bcc":"other@my-company.com",
        "subject":"test",
        "body":"Hello World"
    }
}
  • CALENDAR
{
    "type":"CALENDAR",
    "calendar":
    {
        "title":"AGM",
        "location":"Unitag - 67 allées Jean Jaurès 31000 Toulouse",
        "start":"10/02/2015 09:00",
        "end":"10/02/2015 12:00"
    }
}
  • CALL
{
    "type":"CALL",
    "call":
    {
        "phone":"+33970805343"
    }
}

Design object

The design object contains all the graphical properties and modifications that define and can be applied to a QR Code image. Its structure is the following:

{
    "owner":20,
    "color":"007FB6",
    "modules":
    {
        ...
    },
    "eyes":
    {
        ...
    },
    "logo":
    {
        ...
    },
    "background":
    {
        ...
    },
    "quietZone":true
}

Only owner, color, modules and eyes are mandatory. By default there is no logo, no background and quietZone is set to true.

  • MODULES

  • EYES

  • LOGO Use the logo object to add an image on the QR Code. Structure:

{
    "url":"https://static-unitag.com/file/freeqr/671f9c9974bf42694ee4eca41be358b5.jpg",
    "width":"0.2",
    "height":"0.2",
    "leftOffset":"0.4",
    "topOffset":"0.6",
    "excavate":"true"
}
    + `url`: is the only mandatory field. It should contain a URL where the logo image can be found. By default the logo is placed at the center of the image and `excavate` is set to true.
    +  `width`: used to specify the width of a logo when not using the default positioning. It works like a percentage of the QR Code image width.
    +  `height`: used to specify the height of a logo when not using the default positioning. It works like a percentage of the QR Code image height.
    +  `leftOffset`: used to specify where to place the logo from the left edge when not using the default positioning. It works like a percentage of the QR Code image width.
    +  `topOffset`: used to specify where to place the logo from the top edge when not using the default positioning. It works like a percentage of the QR Code image height.
    +  `excavate`: if set to true (by default), the QR Code modules behind the logo are removed and this makes it look better. If set to false, the logo is simply placed on the QR Code but can partially hide some of the modules.

+ BACKGROUND
`background` can be set to either define a background color or a shadow.
Background color:
```json
{
    "color":"FFCC00"
}

QR Code shadow:

{
    "shadow":
    {
        "color":"BDBDBD",
        "strength":"M"
    }
}

There are 3 different values for the field strength field which defines the strength of the shadow: L (light), M (medium), S (strong).
